════════════════════════════════════════════════════════════════
VISUAL VOCABULARY
════════════════════════════════════════════════════════════════
LONG-SIDE ELEVATION:
  Thick vertical rectangle         → POTEAU
  Thick horizontal at top          → SABLIERE
  Thick intermediate horizontal    → PANNE
  Single diagonal in panel         → PALÉE DE STABILITÉ
  Two diagonals crossing (X)       → CONTREVENTEMENT CVT
  Element with beveled end         → JARRET

ROOF PLAN:
  Large longitudinal beams         → TRAVERSE
  Diagonals in horizontal plane    → POUTRE AU VENT
  Short corner diagonals           → DRETELLES
  Thin perpendicular bars          → LIERNE (round bar D14 typ.)
  Regular parallel grid            → PANNES COURANTES
  Central ridge beam               → PANNE FAÎTIÈRE
  Low perimeter beam               → SABLIERE

GABLE ELEVATION:
  Main vertical façade columns     → POTEAU PIGNON
  Small intermediate columns       → POTELET
  Dense horizontal/vertical grid   → LISSES + MONTANTS BARDAGE

ASSEMBLY DETAILS:
  Flat rectangle + thickness cote  → PLATINE (PL) or Tôle Noire (TN)
  Triangular/trapezoidal plate     → GOUSSET
  Small angle cleat fixing purlin  → ÉCHANTIGNOLE
  Threaded rod noted M20/M24/ø20   → TIGE D'ANCRAGE
  Vertical plate inside web        → RAIDISSEUR

DO NOT EXTRACT:
  ─ ─ ─   dimension/axis lines
  ←────→  dimension lines with arrows
  ○  ⊕    bolt/weld symbols
  B1, P1  reference tags (note them, never extract as profiles)
  NERVESCO / sheeting / colliers / visserie

════════════════════════════════════════════════════════════════
FILES = VIEWS, NOT ELEMENTS (CRITICAL — source #1 of duplicates)
════════════════════════════════════════════════════════════════
In Moroccan charpente plans, "File 1", "File 2", "File .1", "File A"
are VIEWS of the same building — NOT different elements.

  "File 1"      = interior frame elevation (long-pan, side view)
  "File 2"      = adjacent frame — usually SAME profile type
  "File .1"     = gable elevation (perpendicular end view)
  "File A–C"    = axis-numbered elevations — may be identical

DEDUPLICATION PROCESS — apply before extracting:
  1. Identify each File's type: long-pan / pignon / toiture
  2. Group Files of the same type together
  3. For each group: count elements ONCE
  4. Set quantity = count × number of identical Files/bays
  5. ALWAYS add quantity_note: "File 1 à 7 — 7 portiques identiques"

  ✓ File 1 + File 2 show same IPE400 at same position
    → ONE entry, quantity multiplied, quantity_note required
  ✓ File 1 (long-pan) + File .1 (pignon) show IPE400
    → TWO entries: POTEAU (long-pan) vs POTELET (pignon)
  ✗ NEVER create separate entries for the same element seen in N files

════════════════════════════════════════════════════════════════
CVT COUNTING RULES (cornières de contreventement)
════════════════════════════════════════════════════════════════
CVT visual recognition:
  → THIN oblique lines (NOT thick rectangles like beams)
  → ALWAYS in PAIRS forming an X in a panel
  → Thinner than columns and beams visually
  → Sometimes drawn in different color (pink/magenta in CAO exports)
  → Annotated: "L70*7", "L50*5", "CVT", "Croix de Saint-André"

COUNTING RULE — ONE X = TWO cornières:
  → Count X crosses visible in the view
  → quantity = number_of_X_crosses × 2
  → quantity_note: "N panneaux × 2 cornières par croix"

CVT DEDUPLICATION BETWEEN VIEWS:
  → X in long-pan + X in roof plan = TWO DIFFERENT entries
    (vertical façade CVT ≠ horizontal roof CVT)
  → X in File 1 + X in File 2 at same panel = SAME element
    → ONE entry, quantity multiplied across bays

LENGTH RULE — ZERO AMBIGUITY
════════════════════════════════════════════════════════════════
CAS 1 — Explicit cote on element or dimension line:
  → length_m      = cote in metres (4000mm → 4.0)
  → length_source = "explicit_dimension"

CAS 2 — Building span visible but NO explicit cote on the piece:
  → PANNE, TRAVERSE, SABLIÈRE, LISSE, LIERNE:
    MUST use building span as cut length
    → length_m      = span in metres
    → length_source = "span_used_as_cut_length"
  → All other elements:
    → length_m      = null
    → length_source = "span_only_not_applicable"

CAS 3 — No dimension visible:
  → length_m      = null
  → length_source = "no_dimension_visible"

ABSOLUTE: length_m always in METRES.
ABSOLUTE: never invent a length — null beats fabrication.
USE building_dimensions.span_m and bay_width_m (extracted in PASS1)
to populate CAS 2 lengths — never read the same span twice.

DESIGNATION NORMALIZATION — READ BEFORE EXTRACTING
════════════════════════════════════════════════════════════════
Before extracting any profile, normalize its designation using
the tables below. Always output the NORMALIZED form in the
"designation" field.

DIAMETER / ROUND BARS:
  Input variants:  D14  Ø14  ø14  ∅14  φ14  RD14  R14  ROND 14  rd14
  Normalized to:   "ø14"
  Rule: strip all prefix variants, keep lowercase ø + number
  Examples: D20 → ø20 / RD12 → ø12 / ROND 24 → ø24

THICKNESS (used in platines, goussets, raidisseurs):
  Input variants:  EP8  ep8  E=8  e=8  Ep:8  Ep.8  ép.8  8mm  EP 8
  Normalized to:   ep value as integer in dims_mm.ep field
  Rule: extract numeric value only → dims_mm.ep = 8
  Never write "EP8" in designation — use dims_mm.ep instead

PLATINES / FLAT PLATES:
  Input variants:  PL  TN  PLT  TP  PLAT  T.N.  TOLE  FER PLAT
  Normalized to:
    - Structural plate (3 dims): "TN ep*larg*long"  e.g. "TN 20*300*450"
    - Flat bar (linear):         "PL larg*ep"        e.g. "PL 150*6"
  Rule: 3 dimensions → TN format / 2 dimensions → PL format
  Examples:
    PLT 20*300*450  → "TN 20*300*450"  (category: platine)
    TP 25*450*450   → "TN 25*450*450"  (category: platine)
    FER PLAT 150*6  → "PL 150*6"       (category: autre, linear)
    T.N. 8*220*180  → "TN 8*220*180"   (category: gousset if triangular)

SQUARE / RECTANGULAR TUBES:
  Input variants:  TC  TU  TUBE-C  Tube Carré  □  SHS  TUBE  T.C.
  Normalized to:   "TC larg*ep"  e.g. "TC 40*2"  or "TC 80*80*4"
  Rule: always prefix with TC + dimensions
  Examples: TUBE-C 40*2 → TC 40*2 / □80*4 → TC 80*4 / SHS 100*5 → TC 100*5

ANGLE SECTIONS (cornières):
  Input variants:  L  ∟  LD  COR  Cor.  Cornière  CORN
  Normalized to:   "L larg*larg*ep"  e.g. "L 70*70*7"
  Rule: always prefix with L + dimensions
  Examples: ∟70*7 → L 70*70*7 / COR 50*5 → L 50*50*5 / LD 80*8 → L 80*80*8

BOLTS (do NOT extract as profiles — note only):
  Input variants:  M16  Ø16 (in bolt context)  BL16  Bls M16  Vis M16
  Rule: bolts are NOT structural profiles
    → If seen in detail: note in detail_ref field of associated platine
    → Never create a profile entry for a bolt
  Bolt context clues: appears near ○ or ⊕ symbols, near "CL6.8" or "CL8.8"

I-BEAM PROFILES:
  Input variants:  IPE400  IPE 400  ipe400  I.P.E 400
  Normalized to:   "IPE400"  (no space, uppercase)
  Same rule for: HEA, HEB, HEM, UPN, UAP, IPN, IPE

FINAL CHECK before outputting JSON:
  □ All ø/D/RD/ROND variants → ø+number
  □ All TC/TU/TUBE-C/□ variants → TC+dims
  □ All L/∟/COR variants → L+dims
  □ All platine variants (3 dims) → TN format
  □ All flat bar variants (2 dims) → PL format
  □ No bolt entries in profiles[]
  □ All profile names uppercase and consistent

COMPUTATION RULES
════════════════════════════════════════════════════════════════
Linear profiles (Standard):
  poids_total_kg = poids_lineaire_kg_m × length_m × quantity

FER PLATS (PL, FER PLAT):
  Identification: PL150x6, PL 150x6, PL150*6 (150=width mm, 6=thickness mm)
  Priority 1: Use catalogue weight if available.
  Priority 2: poids_lineaire_kg_m = larg_mm × ep_mm × 0.00785
  poids_total_kg = quantity × length_m × poids_lineaire_kg_m

Plates (Platines, goussets, échantignolles, raidisseurs):
  Format: PLT300x250x10 or TN 10*250*300
  poids_total_kg = (long_mm/1000) × (larg_mm/1000) × (ep_mm/1000) × 7850 × quantity

summary.total_weight_kg = sum of all non-null poids_total_kg
summary.extraction_confidence_avg = mean of all confidence values
